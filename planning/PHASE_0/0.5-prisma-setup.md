# Task 0.5 — Prisma Init + Schema + Migrate

## Overview

Install Prisma, initialize configuration, push the schema to Supabase, and create the Prisma client singleton.

**Estimated Time:** 3 hours
**Priority:** P1 (Blocker)
**Dependencies:** Task 0.4 completed (schema designed)

---

## Prerequisites

- [ ] Task 0.4 completed (schema.prisma created)
- [ ] `.env.local` has valid `DATABASE_URL`
- [ ] Supabase project is running

## Steps

### Step 1: Install Prisma

```bash
cd /Users/girlygallardo/Documents/Projects/legacyscale-saas

pnpm add prisma @prisma/client
pnpm add -D prisma
```

### Step 2: Initialize Prisma (Skip if schema.prisma exists)

If `prisma/schema.prisma` doesn't exist yet:

```bash
pnpm prisma init
```

This creates:
- `prisma/schema.prisma`
- Updates `.env` (we use `.env.local` instead)

### Step 3: Verify Schema File

Ensure `prisma/schema.prisma` contains the full schema from Task 0.4.

The datasource block should be:

```prisma
datasource db {
  provider  = "postgresql"
  url       = env("DATABASE_URL")
  directUrl = env("DATABASE_URL")
}
```

### Step 4: Push Schema to Database

```bash
pnpm prisma db push
```

This will:
- Create all tables in Supabase
- Create all enums
- Set up foreign key relationships

Expected output:
```
🚀  Your database is now in sync with your Prisma schema.
```

### Step 5: Generate Prisma Client

```bash
pnpm prisma generate
```

This creates the TypeScript client in `node_modules/@prisma/client`.

### Step 6: Create Prisma Client Singleton

Create `lib/prisma.ts`:

```bash
cat > lib/prisma.ts << 'EOF'
import { PrismaClient } from '@prisma/client'

const globalForPrisma = globalThis as unknown as {
  prisma: PrismaClient | undefined
}

export const prisma = globalForPrisma.prisma ?? new PrismaClient({
  log: process.env.NODE_ENV === 'development' ? ['query', 'error', 'warn'] : ['error'],
})

if (process.env.NODE_ENV !== 'production') globalForPrisma.prisma = prisma

export default prisma
EOF
```

### Step 7: Create Seed Script

Create `prisma/seed.ts`:

```bash
cat > prisma/seed.ts << 'EOF'
import { PrismaClient } from '@prisma/client'

const prisma = new PrismaClient()

  console.log('🌱 Starting seed...')

      orderIndex: 0,

  const chapter = await prisma.chapter.upsert({
    where: { id: 'sample-chapter-1' },

    where: { id: 'sample-lesson-1' },

    create: {

      chapterId: chapter.id,

      orderIndex: 0,

  const question2 = await prisma.quizQuestion.upsert({
    where: { id: 'sample-question-2' },
    update: {},
    create: {
      id: 'sample-question-2',
      lessonId: quizLesson.id,
      questionText: 'Marketing fulfillment arbitrage is a valid business model.',
      type: 'TRUE_FALSE',
      options: ['True', 'False'],

  const resourceLesson = await prisma.lesson.upsert({
    where: { id: 'sample-lesson-3' },

    create: {
      id: 'sample-lesson-3',
```

### Step 11: Test Prisma Client in API Route

Update `app/api/health/route.ts`:

```bash
cat > app/api/health/route.ts << 'EOF'
import { prisma } from '@/lib/prisma'
import { NextResponse } from 'next/server'

export async function GET() {
  try {
    // Test database connection with Prisma
    const userCount = await prisma.user.count()
    const courseCount = await prisma.course.count()

    return NextResponse.json({
      status: 'ok',
      message: 'Database connection successful',
      data: {
        users: userCount,
        courses: courseCount,
      },
      timestamp: new Date().toISOString()
    })
  } catch (error) {
    console.error('Database error:', error)
    return NextResponse.json({
      status: 'error',
      message: error instanceof Error ? error.message : 'Unknown error',
      timestamp: new Date().toISOString()
    }, { status: 500 })
  }
}
EOF
```

## Troubleshooting

### "Can't reach database server"
- Check `DATABASE_URL` in `.env.local`
- Verify Supabase project is running
- Check for typos in connection string

### "P1001: Can't reach database server"
- The database might be paused (free tier)
- Go to Supabase dashboard and unpause

### "Prisma schema validation error"
- Run `pnpm prisma format` to fix formatting
- Check for syntax errors in schema.prisma

### "tsx: command not found"
```bash
pnpm add -D tsx
```

### Seed fails with duplicate key
- Data already exists from previous run
- Run `pnpm db:reset` to clear and reseed
