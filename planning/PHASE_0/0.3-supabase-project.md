Create a Supabase project, configure authentication settings, and set up environment variables for the Legacy Scale platform.

## Steps

### Step 1: Create Supabase Project

1. Go to https://supabase.com/dashboard
2. Click **"New Project"**
3. Fill in:
   - **Name:** `legacyscale-dev` (or `legacyscale-prod` for production)
   - **Database Password:** Generate a strong password (save this!)
   - **Region:** Choose closest to your users (e.g., `us-east-1`)
4. Click **"Create new project"**
5. Wait for project to be provisioned (~2 minutes)

### Step 2: Get API Keys

1. In Supabase dashboard, go to **Settings → API**
2. Copy these values:

| Key | Location |
|-----|----------|
| Project URL | `https://xxxxx.supabase.co` |
| anon/public key | Under "Project API keys" |
| service_role key | Under "Project API keys" (keep secret!) |

### Step 3: Get Database Connection String

1. Go to **Settings → Database**
2. Find **"Connection string"** section
3. Select **"URI"** tab
4. Copy the connection string (format: `postgresql://postgres:[PASSWORD]@...`)
5. Replace `[YOUR-PASSWORD]` with your database password

### Step 4: Update .env.local

```bash
cd /Users/girlygallardo/Documents/Projects/legacyscale-saas

```

**Replace:**
- `YOUR_PROJECT_ID` with your actual Supabase project ID
- `your_anon_key_here` with your anon key
- `your_service_role_key_here` with your service role key
- `YOUR_PASSWORD` with your database password

### Step 5: Install Supabase Packages

```bash
pnpm add @supabase/supabase-js @supabase/ssr
```

### Step 6: Create Supabase Client Utilities

Create `lib/supabase/client.ts` (Browser client):

```bash
mkdir -p lib/supabase

cat > lib/supabase/client.ts << 'EOF'
import { createBrowserClient } from '@supabase/ssr'

export function createClient() {
  return createBrowserClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!
  )
}
EOF
```

Create `lib/supabase/server.ts` (Server Component client):

```bash
cat > lib/supabase/server.ts << 'EOF'
import { createServerClient, type CookieOptions } from '@supabase/ssr'
import { cookies } from 'next/headers'

export async function createClient() {
  const cookieStore = await cookies()

  return createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    {
      cookies: {
        getAll() {
          return cookieStore.getAll()
        },
        setAll(cookiesToSet) {
          try {
            cookiesToSet.forEach(({ name, value, options }) =>
              cookieStore.set(name, value, options)
            )
          } catch {
            // The `setAll` method was called from a Server Component.
            // This can be ignored if you have middleware refreshing
            // user sessions.
          }
        },
      },
    }
  )
}
EOF
```

Create `lib/supabase/middleware.ts` (Middleware client):

```bash
cat > lib/supabase/middleware.ts << 'EOF'
import { createServerClient } from '@supabase/ssr'
import { NextResponse, type NextRequest } from 'next/server'

export async function updateSession(request: NextRequest) {
  let supabaseResponse = NextResponse.next({
    request,
  })

  const supabase = createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    {
      cookies: {
        getAll() {
          return request.cookies.getAll()
        },
        setAll(cookiesToSet) {
          cookiesToSet.forEach(({ name, value, options }) =>
            request.cookies.set(name, value)
          )
          supabaseResponse = NextResponse.next({
            request,
          })
          cookiesToSet.forEach(({ name, value, options }) =>
            supabaseResponse.cookies.set(name, value, options)
          )
        },
      },
    }
  )

  // Do not run code between createServerClient and
  // supabase.auth.getUser(). A simple mistake could make it very hard to debug
  // issues with users being randomly logged out.

  const {
    data: { user },
  } = await supabase.auth.getUser()

  return { supabaseResponse, user }
}
EOF
```

Create `lib/supabase/admin.ts` (Admin client with service role):

```bash
cat > lib/supabase/admin.ts << 'EOF'
import { createClient } from '@supabase/supabase-js'

// Admin client with service role key - use only on server side
// This bypasses Row Level Security
export function createAdminClient() {
  return createClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.SUPABASE_SERVICE_ROLE_KEY!,
    {
      auth: {
        autoRefreshToken: false,
        persistSession: false
      }
    }
  )
}
EOF
```

### Step 7: Configure Auth Settings in Supabase Dashboard

1. Go to **Authentication → Providers**
2. Ensure **Email** is enabled
3. Configure settings:
   - Enable email confirmations: **OFF** (for MVP simplicity)
   - Secure email change: **ON**
   - Secure password change: **ON**

4. Go to **Authentication → URL Configuration**
5. Set:
   - Site URL: `http://localhost:3000`
   - Redirect URLs: Add `http://localhost:3000/**`

### Step 8: Configure Auth Email Templates (Optional)

1. Go to **Authentication → Email Templates**
2. Customize templates if desired (can do later)

### Step 9: Test Connection

Create a test API route `app/api/health/route.ts`:

```bash
mkdir -p app/api/health

cat > app/api/health/route.ts << 'EOF'
import { createClient } from '@/lib/supabase/server'
import { NextResponse } from 'next/server'

export async function GET() {
  try {
    const supabase = await createClient()

    // Test database connection
    const { data, error } = await supabase
      .from('_test_connection')
      .select('*')
      .limit(1)

    // This will error because table doesn't exist, but it proves connection works
    if (error && !error.message.includes('does not exist')) {
      throw error
    }

    return NextResponse.json({
      status: 'ok',
      message: 'Supabase connection successful',
      timestamp: new Date().toISOString()
    })
  } catch (error) {
    return NextResponse.json({
      status: 'error',
      message: error instanceof Error ? error.message : 'Unknown error',
      timestamp: new Date().toISOString()
    }, { status: 500 })
  }
}
EOF
```

## Troubleshooting

### "Invalid API key"
- Double-check you copied the full key
- Ensure no extra spaces or newlines in `.env.local`
- Restart dev server after changing env vars

### "Connection refused"
- Check project is active in Supabase dashboard
- Verify the project URL is correct
- Check your internet connection

### "CORS error"
- Add your local URL to Supabase URL Configuration
- Go to Authentication → URL Configuration → Redirect URLs

### Database connection issues
- Verify the password is URL-encoded if it contains special characters
- Check the connection string format
