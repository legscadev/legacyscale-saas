# Task 0.7 — Deploy to Vercel

## Overview

Connect GitHub repository to Vercel, configure environment variables, and deploy the Legacy Scale platform to production.

**Estimated Time:** 1 hour
**Priority:** P1 (Blocker)
**Dependencies:** Tasks 0.1-0.6 completed

---

## Prerequisites

- [ ] GitHub repository created with project code
- [ ] Vercel account (create at https://vercel.com)
- [ ] All environment variables from `.env.local`

---

## Steps

### Step 1: Push Code to GitHub

If not already done:

```bash
cd /Users/girlygallardo/Documents/Projects/legacyscale-saas

# Initialize git if needed
git init

# Add all files
git add .

# Commit
git commit -m "chore: Phase 0 foundation setup"

# Add remote (replace with your repo URL)
git remote add origin https://github.com/YOUR_USERNAME/legacyscale-saas.git

# Push
git push -u origin main
```

### Step 2: Create Vercel Project

1. Go to https://vercel.com/dashboard
2. Click **"Add New..."** → **"Project"**
3. Select **"Import Git Repository"**
4. Connect your GitHub account if not already connected
5. Find and select `legacyscale-saas` repository
6. Click **"Import"**

### Step 3: Configure Project Settings

On the configuration screen:

| Setting | Value |
|---------|-------|
| Framework Preset | Next.js (auto-detected) |
| Root Directory | `./` |
| Build Command | `pnpm build` (or leave default) |
| Output Directory | `.next` (default) |
| Install Command | `pnpm install` |

### Step 4: Configure Environment Variables

Add these environment variables in Vercel:

```
NEXT_PUBLIC_SUPABASE_URL=https://YOUR_PROJECT.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key
DATABASE_URL=postgresql://postgres:PASSWORD@db.YOUR_PROJECT.supabase.co:5432/postgres

MUX_TOKEN_ID=your_mux_token_id
MUX_TOKEN_SECRET=your_mux_token_secret
MUX_WEBHOOK_SECRET=your_mux_webhook_secret

RESEND_API_KEY=your_resend_api_key

GHL_WEBHOOK_SECRET=your_ghl_webhook_secret

NEXT_PUBLIC_APP_URL=https://app.legacyscale.co
```

**To add each variable:**
1. Click **"Environment Variables"** section
2. Enter variable name
3. Enter variable value
4. Select environments: **Production**, **Preview**, **Development**
5. Click **"Add"**

### Step 5: Deploy

Click **"Deploy"** button.

Wait for deployment (usually 1-3 minutes).

### Step 6: Verify Deployment

1. Vercel will show deployment URL (e.g., `legacyscale-saas.vercel.app`)
2. Visit the URL
3. Check that the home page loads
4. Visit `/api/health` to verify API works

### Step 7: Configure Custom Domain (Optional - Can Do Later)

1. Go to **Project Settings** → **Domains**
2. Add `app.legacyscale.co`
3. Vercel will provide DNS records
4. Add DNS records in your domain registrar:
   - **A Record**: `76.76.21.21`
   - **CNAME**: `cname.vercel-dns.com`
5. Wait for DNS propagation (up to 48 hours, usually faster)
6. Vercel auto-provisions SSL certificate

### Step 8: Update Supabase Redirect URLs

1. Go to Supabase Dashboard → **Authentication** → **URL Configuration**
2. Add to **Redirect URLs**:
   - `https://legacyscale-saas.vercel.app/**`
   - `https://app.legacyscale.co/**` (if using custom domain)
3. Update **Site URL** to production URL

### Step 9: Enable Preview Deployments

Vercel automatically creates preview deployments for PRs. Verify:

1. Go to **Project Settings** → **Git**
2. Ensure **"Automatically deploy"** is enabled
3. Each PR will get a unique preview URL

### Step 10: Set Up Deployment Protection (Optional)

For security before launch:

1. Go to **Project Settings** → **Deployment Protection**
2. Enable **"Vercel Authentication"** for preview deployments
3. Only team members can access preview URLs

## Troubleshooting

### Build fails with "Module not found"
- Check all dependencies are in `package.json`
- Run `pnpm install` locally and commit `pnpm-lock.yaml`

### Build fails with TypeScript errors
- Run `pnpm type-check` locally
- Fix all TypeScript errors before pushing

### Environment variables not working
- Check variable names match exactly (case-sensitive)
- Check no extra spaces in values
- Redeploy after adding variables

### API routes returning 500
- Check Vercel logs: **Deployments** → **[deployment]** → **Functions**
- Verify database connection string is correct

### Custom domain not working
- Check DNS records are correct
- Wait for DNS propagation (up to 48 hours)
- Check Vercel domain status

### Preview deployments not created
- Check GitHub integration is connected
- Check deploy hooks are not paused

---

## Rollback Process

If something goes wrong:

1. Go to **Deployments** tab
2. Find the last working deployment
3. Click **"..."** → **"Promote to Production"**
4. New deployment is live in ~30 seconds
